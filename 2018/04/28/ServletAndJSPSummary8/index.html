Servlet/JSP的集成：MVC 架构

实现 MVC 模式的应用程序由三个模块组成

  • 模型 Model： 负责数据和业务逻辑。
  • 视图 View： 负责应用程序的显式
  • 控制器 Controller： 负责接收用户的输入，并控制 Model 和 View 做出相应的改变

控制器 Controller

在 MVC 框架中，控制器 Controller 是由一个 Servlet 或者过滤器实现的，例如常用的 Web 框架 Struct 1 与 Spring MVC 在它们的 MVC 架构中使用一个 Servlet Controller， 而另一个流行的框架 Structs 2，则是使用过滤器。

视图 View

视图 View 层一般由 JSP 技术实现，例如 Spring MVC 支持许多类型的视图用于不同的表示技术，其中最常用就是 JSP 技术。

模型 Model

模型 Model 可以由 JavaBean、 EJB、 POJO(Plain Old Java Object)实现，其中 POJO 与 JavaBean 比较流行。

在 MVC 模式中的应用程序，每一个 HTTP 请求都必须被定向到控制器中，请求的 URI 告诉控制器要调用那一个 Action，Action 指的是 Web 程序能够执行的一箱操作，与 Action 相关的 Java 对象被称为 Action 对象，一个 Action 类可以服务几个不同的 Action，例如 Structs 2 和 Spring MVC ，或单个 Action ，例如 Struct 1。

控制器会检查 URI 决定调用哪一项 Action ，还会将模型对象保存在一个视图 View 可以访问的对象，通常被称为 VO (Value-Object)。然后控制器使用一个 RequestDIspatcher 跳转到一个 View ( JSP 页面）。 JSP 页面使用 EL 和 JSTL 显示其值。

MVC 模式使得创建和操作数据的代码与表达数据的代码分离。

关键在于使用 RequestDispatcher

Servlet 处理最初的请求，查找数据，并将结果存储在 bean 中，然后请求转发到 JSP 页面来提交最后的结果，JSP页面使用 bean。

Servlet 业务逻辑和数据处理，创建 bean 来表示相应的结果，然后确定哪个 JSP 页面适合表达这些特定的结果，并将请求转发到相应的页面。

使用 RequestDispatcher 实现 MVC

Step1：定义 JavaBean 来表示数据

JavaBean 是 Java 中一种特殊的类。符合以下规范的类称为 JavaBean：

  1. 具有一个 public 的无参构造器。
  2. 可序列化，实现java.io.Serializable接口。
  3. 每个属性都是private的，都有相对应的setter()和getter()方法，JavaBean 的使用者只能通过这两个方法访问属性。

在 Servlet/JSP 中使用的 JavaBean 有两种，一种是表单 FormBean，用于读取请求中的参数， 另外一种主要供数据库、视图 View 使用的 Bean。

映射HTML表单的FormBean

Form 类被映射到一个 HTML 表单，它是 HTML 表单在服务器中的表示法，Form 对象有主要用途是保存用户的输入供服务端处理，例如当用户的输入经过验证不符合要求时，原始表单会读取 Form 类重新显示，即做自动填充。

映射数据库记录的DAO

DAO 全称 Data Access Object ，顾名思义是数据访问对象，是一种访问数据库中的数据的设计模式。使用这种模式，需要为持久化的每一种类型对象都编写一个类，通常类名以 DAO 结尾。

Step2：使用 servlet 处理请求

在定义 JavaBean 之后，接下来就是编写 servlet 读取请求信息，这里回顾一下 servlet 读取请求信息的方式。

原始的方法

最原始的方法是在 servlet 中调用HttpRequest.getParameter()、HttpRequest.getParameterValues等方法，但这些方法返回值类型均为String，需要类型转换，这就导致在servlet中要编写大量的类型转换代码、转换异常捕获，例如，如果你需要获得一个double类型的值，则必须将getParameter返回的值传递给Integer.parseInt，并且将相应的代码放入try catch块，检查NumberFormatException异常。如果要读取很多的参数，则需要不断的编写类似代码，导致过程十分冗长。

Step3：填写 Bean

在 MVC 框架中，servlet 读取请求中的信息一般使用一个表单Form Bean， 读取完来自请求的表单信息后，根据表单信息查询数据库、业务处理，得到的处理后的数据作为一个 Bean 对象。

Step4：结果存储

到目前为止，我们已经读取了请求中的表单信息，读取的数据进过业务处理，并以这些数据创建初始化了一个 Bean 对象，可能还会将这些数据写入数据库。现在我们要将这些 Bean 存储到 JSP 能够访问的地方。

servlet 能够在三个位置存储 JSP 页面所需要的数据，分别是HttpServletRequest、HttpSession、ServletContext，这些存储位置对应着 JSP 技术(useBean、EL、JSTL)中的三个scope属性request、session、application，在JSP中一般有两种方式读取这些数据，一种是使用useBean动作元素，另外一种是使用 EL 表达式。前者在实际开发中很少用了，因此我们只介绍 EL 表达式读取数据。
