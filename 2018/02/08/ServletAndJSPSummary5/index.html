Servlet/JSP学习总结（5） JSTL 库核心标签

Posted on 2018/02/08

JSP 页面中使用 JSTL 目的是为了不希望出现 Java 业务代码，同时弥补 Html 标签的不足。JSTL 是指JSP标准标签库，通过多个标签库来规定其动作指令，其中包括：Core、XML、i18n、Sql、functions 五种标签库，但是除了 Core 标签库，其他的标签库都很少用了，因此我们只关注 JSTL 中的 Core 标签库。

使用环境

  1. 前往下面的网址
    http://archive.apache.org/dist/jakarta/taglibs/standard/binaries/
    选择最新版本的 zip 包下载并解压，到解压后的 lib 文件夹将 jstl.jar和standard.jar复制到项目 WEB-INF/lib 目录下。

  2. 在需要使用 JSTL 的 JSP 页面头部中插入一下代码，引入 Core 标签库。

Core 标签库分类

两种格式： 所有的标签都有两种语法形式，一种包含主体，另一种不包含主体。

下面重点关注 Core 库中比较重要的几类标签

类别 标签 用途
通用动作指令： out 、set 、remove 标签 主要用于操作各种域变量。
条件动作指令： if 、choose 、when 、otherwise 标签 用于控制流程处理。
iterator 动作指令： forEach 、forTokens 标签 用于迭代各种对象或集合。

通用动作指令

语法格式：

使用第一种形式输出Session中的myVar变量的值，其中 EL 表达式获取myVar变量的值。

out 标签相关属性

属性 类型 描述
value Object value= 后面接 EL 表达式。
escapeXml Boolean 规定结果中<，>，'，"和&转换成那种字符实体代码，如<转成&lt;。
defualt Object 默认值

set 标签

语法格式与作用

第一种，设置JavaBean对象属性值，将value赋值给相应 JavaBean 的属性，相当于 setter 方法：target.setProperty(value)。

第二种，创建一个变量并赋值，然后存放在对应的域内，或引用一个存在的域变量。

set 标签的相关属性

属性 类型 描述
value Object 创建的字符串，或要引用的域对象，或新的属性值
var String 创建的域变量名
scope String 对应的域范围
target Object 域对象，必须是JavaBean或java.util.Map对象
property Stirng 域对象的属性

iterator 动作指令

JSTL 提供了两个可以执行 iterator 动作指令的标签： forEach 和 forTakens。

forEach 标签

功能：根据循环条件，遍历对象或集合中的元素。

语法格式

属性

[]表示可选属性，相关属性的类型与描述见下表：

属性 类型 描述 是否支持 EL 表达式
items 任何类型 需要迭代的对象集合 支持
var String 集合中元素的引用 不支持
varStatus String 保存迭代状态与变量名称，值类型为javax.servlet.jsp.jstl.core.LoopTagStatus 不支持
begin int 遍历的开始位置，值大小必须大于或等于 0 支持
end int 遍历的结束位置，值大小必须大于或等于1 支持
step int 遍历的步长，循环的步长 支持

其中javax.servlet.jsp.jstl.core.LoopTagStatus四个状态为：

属性 类型 描述
index int 当前循环的索引值
count int 循环的次数
first boolean 是否为第一个位置
last boolean 是否为最后一个位置
