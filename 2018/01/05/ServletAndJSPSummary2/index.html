Servlet/JSP学习总结（2） JSP

Posted on 2018/01/05

JSP技术

JSP 全称JavaServer Page，是对Servlet技术的补充，可以将常规的、静态的HTML与动态生成的内容混合起来。

Servlet的缺陷与JSP的优点

Servlet 擅长于业务逻辑或数据处理，而在需要生成输出的时候，Servelt有几个缺陷无法克服：

  • Servlet 中的HTML的编写和维护十分困难
    在Servlet中编写HTML，必须使用大量PrintWriter.println()语句，对字符串进行拼接，当页面比较的复杂时，编写起来十分的困难。另外使用这种方式，所有文本和HTML标签必须硬编码，一旦对表示层做微小的该改动时，例如，修改背景颜色，也需要重新编译。
  • 非Java开发人员难以处理这些HTML
    将HTML嵌入到Servlet中，不了解Java语言的Web开发人员难以检查和更改这些HTML。

JSP的出现是为了拟补Servlet表示上的补足，使用JSP有以下几个优点：

  • JSP中的HTML的编写和维护更加的简单。
  • 可以使用标准的网站开发工具，例如HTML工具Dreamweaver。
  • 将静态HTML和动态内容进行了分离。

JSP生命周期

  • 转换阶段
    这个阶段只出现在页面创建或者页面被修改后，客户端的第一次请求。此时JSP容器将JSP页面转换成Servlet，这种转化只是简单地将所有模板文本改用println()语句，并且将所有的JSP元素转化成Java代码。此时的JSP文件为someJSP.java

  • 编译阶段
    这个阶段同样只出现在页面创建或者页面被修改后，客户端的第一次请求。JSP容器编译servlet源文件，生成servlet类。此时JSP文件为someJSP.class

  • 加载阶段
    客户端对JSP页面进行第一次请求时，JSP容器会将编译好的class加载服务器的内存中，并且常驻内存，直至JSP页面被修改。

  • 初始化阶段
    该阶段紧跟在加载阶段之后，同样也发生在客户端对JSP页面进行第一次请求的时候，JSP容器将调用jspInit()，进行初始化工作。

  • 执行阶段
    当每次客户端发出请求时，常驻服务器内存的servlet实例将调用_jspService()处理来自客户端的请求，并生成相应的响应格式，如HTML，返回给客户端。

  • 销毁阶段
    调用与JSP对应的servlet实例的销毁方法，然后销毁servlet实例；

JSP九种内置对象

JSP内置对象是JSP容器为每个页面提供的Java对象，开发者可以直接使用它们而不用通过显示声明，也成预定义变量。

属性 描述
request request对象是javax.servlet.http.HttpServletRequest类的实例，当客户端请求一个JSP页面时，JSP引擎就会制造一个新的request来代表这个请求。这个对象提供一系列的方法来获取http头信息，cookied，HTTP方法等等。
response response对象是javax.servlet.http.HttpServletResponse类的实例，JSP引擎在创建request对象同时也会同时创建用于响应这个客户端的response对象
out out对象是javax.servlet.jsp.JspWriter类的实例。
session session对象是javax.servlet.http.HTTPSession类的实例，用来跟踪各个客户端请求间的会话。
application application对象直接包装了servlet的ServletContext类的对象，是javax.servlet.ServletContext类的实例。在整个JSP页面的生命周期中都代表着这个JSP页面，经历初始化、执行、销毁等时期。
config 不常用
pageContext 用来代表整个JSP页面，存储了request对象和response对象的引用。
page 页面实例的引用，相当于this。
Exception 包装了先前页面中跑出的异常信息。

request对象

客户端所有的请求信息都封装在request对象，服务端只有通过它才能了解到客户的需求，根据需求做出响应。

常用的方法如下：

方法名 描述
Cookie[] getCookies() 返回客户端所有的Cookie的数组
Enumeration getHeaderNames() 返回所有HTTP头的名称集合
String getHeader(String name) 返回HTTP头中名为name参数的值
HtppSession getSession() 返回request对应的session对象，若没有则创建一个
String getCharacterEncoding() 返回request的字符编码集名称
String getMethod() 返回request中的HTTP方法，如GET，POST或PUT
String getParameter(String name) 返回request中参数名为name的值，若不存在则返回null
String getRemoteAddr() 返回客户端的IP地址
String getRemoteHost() 返回客户端的完整名称
String getServletPath() 返回所请求的servlet路径
实例3：通过request对象获取客户端请求的HTTP信息头

下面我们使用request对象中的方法来获取请求HTTP信息头，通过request.getHeaderNames()获取Enumeration对象，然后遍历Enumeration对象，得到Header参数名，通过参数名使用request.getHeader(String name)获取信息头中对应参数值以html表格形式输出。

response对象

Response对象主要将JSP容器处理后的结果传回客户端，可以通过response变量设置HTTP的状态和对客户端发送数据，如Cookie、HTTP文件头信息。

实例4 模拟数字时钟

我们可以通过给request.setIntHeader(String name, int value)传入Refresh和5参数，使得jsp页面每个五秒刷新一次时间值，达到模拟数字时钟的效果。

Note

Q1 使用<jsp:include>与<%@ include file="relative url"%>都能导入外部内容？哪种方式比较好？

<jsp:include>导入文件内容的方式比较好。这是因为：

  • 使用<%@ include file="relative url"%>的页面要比是用<jsp:include>难维护得多。

Q2 请求转发与请求重定向的区别？

  • 请求转发：
    请求转发，即request.getRequestDispatcher().forward()，是一种服务器的行为，客户端只有一次请求，服务器端转发后会将请求对象保存，地址栏中的URL地址不会改变，得到响应后服务器端再将响应发给客户端；

  • 请求重定向：
    请求重定向，即response.sendRedirect()，是一种客户端行文，从本质上讲等同于两次请求，前一次请求对象不会保存，地址栏的URL地址会改变。
