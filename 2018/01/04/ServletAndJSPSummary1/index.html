Servlet/JSP学习总结（1）Servlet

Posted on 2018/01/04

Servlet

什么是 Servlet？

Servlet是Server applet 的缩写，意思是运行在服务端的Java程序。

从作用上看，Servlet是一个中间层，负责接受来自Web浏览器或其他HTTP客户程序的请求，连接HTTP服务器上的数据库或应用程序，处理这些请求并返回响应。更具体一点：

  • 读取客户端发送的显式数据和隐式数据：显式数据有可能来自用户提交的HTML表单信息。隐式数据通常指的是HTTP信息，包括Cookie、浏览器所能识别的媒体类型等。
  • 生成结果：生成结果的过程可能需要访问服务端的数据库，也有可能需要调用Web服务。
  • 给客户端发送显式数据和隐式数据：这里的显式数据通常指文档，文档的格式是多样的，包括文本（HTML或XML）、二进制图（GIF图）等，最常用的格式是HTML

从具体实现细节上看：

  • 它符合J2SE语法规则，是常规的Java代码，但有着不同于J2SE的规范。
  • Servlet在Servlet容器（如Tomcat）中运行，不能自动运行，Servlet与Servlet容器存在大量的接口，这些接口定义Servlet与Servlet容器的契约，是Servlet技术的核心。
  • Servlet容器会将Servlet类加载到内存中，并在Servlet实例中调用特定的方法。

Servlet 包结构

Servlet API中有4个Java包，包括：

  • javax.servlet。包含定义了Servlet容器与Servlet之间契的类以及接口。
  • javax.servlet.http。 包含了定义HTTP Servlet与Servlet容器之间契约的类和接口。
  • javax.servlet.annotation。包含了对Servlet、Filter、Listener进行标注的注解。
  • javax.servlet.descriptor。包含为Web应用程序的配置信息提供编程式访问的类型

其中我们重点关注两个包javax.servlet与javax.servlet.http。

Servlet 接口

Servlet 接口定义了五个方法：

  • void init(ServletConfig config)
  • void service(ServletRequest request, ServletResponse response)
  • void destroy()
  • ServletConfig getServletConfig()。返回Servlet容器传给init方法的ServletConfig。
  • String getServletInfo()。返回Servlet的描述。

Servlet 生命周期

init、service、destroy方法属于Servlet生命周期方法。

  • init。顾名思义，对Servlet初始化。当第一次请求某个Servlet时，Servlet容器将调用这个方法，并传递一个ServletConfig，我们可以在这个方法内编写一些应用程序初始化相关的代码，对这个Servlet进行初始化。init方法在整个生命周期中只会被调用一次。
  • service。每一次请求时，Servlet容器都将会调用这个方法，在这里编写要Servlet完成的代码。
  • destroy。当要销毁Servlet时，Servlet容器就会调用这个方法，通常会发生在卸载应用程序或关闭Servlet容器时。可以在这里编写一些资源清理的相关代码。

ServletRequest

对于每一次HTTP请求，Servlet容器都会创建一个实现了ServletRequest接口的实例，并将它传给Servlet的service()方法。这个ServletRequest实例封装着所有这次HTTP请求的信息，包括parameter、value、atrributes和inputStream等。
下面是ServletRequest接口常用方法：

  • public int getContentLength()，返回请求主体中的字节数，若不知道该字节的长度，则返回-1
  • public String getContentType()，范湖请求主题的MIME类型，若不知道类型则返回-1
  • public String getParameter(String name)，返回指定参数名为name的值，通常用于获取返回一个HTML表单域的值和查询字符串的值。
  • public String getProtocol()，返回HTTP请求的协议名称和版本号。

ServletResponse 接口

在调用servlet的service()方法之前，Servlet容器会创建一ServletResponse实例，并作为service()方法的第二参数传递给service()方法。可以在service方法中通过对ServletResponse实例的编写，封装信息以响应浏览器。

如实例1中，ServletResponse的getWriter()方法返回一个java.io.PrintWriter对象，通过这个对象编写响应浏览器的HTML信息。这里注意一下PritWriter的编码格式默认为ISO-8859-1，若想要向浏览器输出汉字，则要通过ServletResponse.setCharacterEncoding(String charset)方法将编码格式设置为UTF-8。

ServletContext

ServletContext用来表示一个Servlet应用程序，也就是说每个Web应用程序只有一个ServletContext。

可以通过ServletConfig调用getServletContext方法以获得ServletContext。

ServletContext主要用于动态注册Web对象。

GenericServlet 抽象类

GernericServlet抽象类实现了Servlet接口与ServletConfig接口，为Servlet接口中的所有方法提供了默认实现，我们不用一一实现一些不需要重载的方法了。

虽然GenericServlet抽象类相比与Servlet使用起来更加的方便，但现实的应用程序中并不常用，使用更多的是接下来要介绍的HTTPServlet。

HTTP Servlet

现实中客户端与服务端之间的交互几乎是建立在HTTP协议上，因此我们编写的Servlet程序也要使用到HTTP协议，javax.servlet.http包含了编写Servlet应用程序的类和接口，包内主要的类型如下

  • Interfaces:
    • HttpServletRequest extendsjavax.servlet.ServletRequest
    • HttpServletResponse extendsjavax.servlet.ServletResponse
    • HttpSession
  • Classes:
    • Cookie
    • HttpServletextendsjavax.servlet.GenericServlet

HttpServlet 抽象类

Servlet主要通过覆盖Servlet.service()方法，来处理和响应一次请求不同。HttpServlet通常不需要再覆盖service()方法，而是覆盖doGet()或doPost()方法，少数情况下还会覆盖HTTP的doPut()、doHead()等方法。

与Servlet.service()传入的参数类似，doGet()等其他方法，接受两个实现了HttpServletRequest和HttpServletResponse接口的对象，这两个接口是分别继承了ServletRequest和ServletResponse接口。

HttpServletRequest接口

HttpServletRequest表示Http环境下的servlet请求。继承ServletRequest接口，并且添加了以下的方法。

  • String getContextPath()，返回表示请求context的请求URI部分
  • Cookie[] getCookies()，返回Cookie对象数组
  • String getHeader(String name)，返回HTTP head值
  • String getMethod()，返回此次请求的HTTP方法的值
  • String getQueryString()，返回此次请求URL中的查询字符串
  • HttpSession getSession()，返回此次请求的session对象

HttpServletResponse接口

HttpServletResponse表示HTTP环境下的servlet响应。继承ServletResponse接口，有如下主要的方法。

  • void addCookie(Cookie cookie)，给这个响应对象添加cookie
  • void addHeader(String name, String value)，给这个相应对象添加标头
  • void sendRedirect(String location)，将浏览器重定向到location位置

Note

Q1: Servlet、GenericServlet和HttpServerlet之间的关系？

从作用上看，它们都是定义一个Servlet的方式。

  • Servlet是一个接口，主要包含着三个生命周期函数init()、service()、destroy()。
  • GenericServlet是实现Servlet接口的抽象类。
  • HttpServlet继承了GenericServlet抽象类，功能最为强大，现实开发中一般通过继承一个HttpServlet抽象类来定义一个Servlet具体类。

从处理客户端发来的请求上看，

  • GenerServlet主要通过重载service()方法，接受ServletRequest、ServletResponse两个参数，处理请求和发送响应。
  • HttpServlet不使用重载service()方法，而是使用doMethod方法，这里的Method是HTTP协议对应的请求方式，如doGet、doPost等方法。我们通过Tomcat的HttpServlet源码更加深入的了解这两者的异同（注：源码基于Tomcat 9.0.6)

通过源码图，我们可以发现，HttpServlet重载了继承GenericServlet抽象类的service()方法，首先将ServletRequest与ServletResponse转换为HttpServletRequest与HttpServletResponse，然后调用自身的protected版本的service()方法，并传递转换后的两个参数。

在protected版本的service()方法内，不直接处理HttpRequest和HttpResponse对象，继续将这两个对象往下传递给相应的doMethod方法。因此我们可以得出HttpServlet实际上是通过doMethod方法处理来自客户端的请求。
