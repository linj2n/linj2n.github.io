【JavaScript语言精粹】读书笔记

Posted on 2018/06/20

语言中的部分特性糟糕带来的麻烦远远地超过它们的价值，例如，因为规范不完善产生了可移植性的问题；一些特性会导致生成那一理解和修改的代码或使得代码风格过于复杂并且容易出错。JavaScript 鸡肋的部分的比重超出了预料，但好在 JavaScript 有一些非常精华的部分，本书旨在揭示 JavaScript 中的精华，同时会偶尔警告要避免鸡肋的部分。

1. Good Part

1.1 Why JavaScript

  • Web 已经变成了一个重要的应用开发平台，而 JavaScript 是 Web 浏览器的语言，是世界上最被轻视的语言之一。很大部分的原因在于浏览器的 API 和 DOM 内容的糟糕导致 JavaScript 语言被误解。
  • 拥有极强表达能力的语言

1.2 Analyzing JavaScript

JavaScript 好的一部分：

  • 函数、弱类型、动态对象与对象字面量表示法

JavaScript 差的一部分：

  • 全局变量的编程模型

函数

  • JavaScript 的函数是基于词法作用域的顶级对象。即函数是根据词法来划分作用域的，而不是动态地划分作用域。

  • JavaScript 是第一个称为主流的 lambda 语言。

弱类型

  • 大部分编程语言都流行要求强类型，原理在于强类型允许编译器在编译时检测错误，越早检测和修复错误，付出的代价越小，但事实上强类型并不会让你的测试工作轻松。

  • 弱类型比较自由，无序建立复杂的类层次。

对象字面量表示法

  • JavaScript 通过列出对象的组成部分，因此能够简单地创建一个对象。
  • 这种表示法促使了 JSON 格式的产生。

原形继承

  • 原型继承是 JavaScript 中的一个有争议的特性。即JavaScript 有一个无类别（Class-free）的对象系统。
  • 对于习惯使用类去创建对象的程序员来说，需要习惯原型继承的特性。

3 对象

除了简单类型：数字、字符串、布尔值、null 和 undefined 值之外，JavaScript中的其他所有值都是对象，如函数、数组和正则表达式等。对象通过引用来传递，它们永远不会被拷贝。

3.1 对象字面量

  • 对象是属性的容器，属性拥有名字与值。属性的名字可以包括空字符串在内的任意字符串。属性值可以是出 underfined 值之外的任何值。
  • 对象是无类别（class-free），因此对新属性的名字和值没有约束。
  • 对象字面量可以出现在任何允许表达式出现的地方。

对象字面量提供了一种非常方便地创建新对象值的表示法；一个对象字面量就是包围在一对花括号中的零或多个Key/Value对。

  • 如果属性名是一个合法的JavaScript 标识符且不是保留字，并不强制要求用“”（引号）括住属性名，因此first-name必须用引号括住（因为含有-字符），若是first_name则不需要。

3.2 检索

检索方式

检索的对象中包含的值的方式有二：

  • 在[]后缀中括住一个字符串表达式，检索对象中包含的值。
  • 直接使用.检索；前提条件：字符串表达式是一个String literal，并且是合法的标识符。这种方式更加紧凑、可读性更好。

返回结果

若尝试检索一个并不存在的成员元素的值，将返回一个 underfined 值。可以使用||运算符填充默认值。

尝试检索一个 undefined 值会导致 TypeError 异常。可以通过 && 运算符来避免错误。

3.3 更新

3.4 原型

每一个对象都存在一个连接它的原型，对象可以继承这个原型的所有的属性（Prototype）；对于所有对象字面量创建的对象，都存在连接它们的原型Object.prototype(来自 JavaScript 标准)。

委托（delegation）

对象在更新（update）时，不影响与原型的连接；原型的连接只有在检索值（retrieval）时才会被用到，如果我们尝试去获取某个对象的某个属性值时，且该对象没有该属性名，那么 JavaScript 会试着从原型对象中获取属性值。如果这个原型对象也没有该属性，那么再从它的原型中寻找，以此类推，直到该过程最后到达终点 Object.prototype，若最终检索失败，则会返回undefined。这一过程称为委托(delegation)。

原型的动态更新

原型关系是一中动态的关系，当我们添加一个新的属性到原型中，该属性就会立即对所有基于该原型创建的对象可见。

3.5 反射

4 Function

4.1 Function Object

函数是对象，而对象是一个 Key/Value 对的集合并拥有一个连到原型对象的隐藏连接。对象字面量产生的对象连接到的是 Object.protoype，函数对象连接到的 Funtion.prototype 。

因为函数是对象，因此可以拥有以下特性：

  • 函数可以存放在变量、对象和数组中；
  • 函数可以被当做参数传递给其他函数，也可以再返回函数；
  • 函数可以拥有方法。

4.2 Function Literal

函数字面量可以出现在任何允许表达式出现的地方，同时函数也可以被定义在其他函数中。一个内部函数自然可以访问自己的参数和变量，也可以访问它被嵌套在其中的那个函数的参数和变量。

通过函数字面量创建的函数对象包含一个连到外部上下文的连接，称为闭包。这是 JavaScript 语言强大表现力的根基。

4.3 Invocation

函数被调用时，除了声明时定义的参数外，每个函数接受两个附加的参数：this与arguments。

第一个参数this的值取决于调用的模式，在 JavaScript 中有四种模式：方法调用模式、函数调用模式、构造器调用模式与apply 调用模式。第二个参数arguments是函数内置数组对象，表示被调用时传入的参数值数组。

参数处理规则： 实际参数（arguments）个数大于形式参数（parameters）个数时，多余的实际参数将会被忽略，反之实际参数不足时，确实的形参值将会被undefined替代。

方法调用模式

描述：当一个函数被保存为对象中的一个属性时，称它为一个方法。JavaScript 将调用对象中的属性方法这一情形归类方法调用模式（The Method Invocation Pattern）。

  • 方法被调用时，this就被绑定到该对象，因此方法能从对象中取值或修改该对象。
  • this到对象的绑定发生在调用的时候，使得函数可以对this高度复用。
  • 通过this可取得它们所属对象的上下文的方法称为公共方法。

函数调用模式

描述：一个函数不属于任何一个对象时，JavaScript 将该函数的调用的情形归类为函数调用模式（The Function Invocation Pattern）。

  • 函数在这种模式被调用时，this 会被绑定到全局对象。这是一个 JavaScript 语言上的一个错误。比较正确的规则应该将this仍然绑定到外部函数的this变量上。
