Spring学习总结（6）使用Spring Data JPA

Posted on 2018/06/20

JPA 全称 Java Persistence API ，是一种能将对象映射到关系数据库中的标准技术。spring-boot-starter-data-jpa POM 提供了一种十分快速方式上手。

spring-boot-starter-data-jpa提供包括以下核心依赖：

  • Hibernate：一个非常流行的 JPA 实现。
  • Spring Data JPA： 帮助开发者更加容易构建基于 JPA 的 Repositories。
  • Spring ORMs：来自 Spring 框架的核心 ORM 支持

实体类

使用传统的方式，JPA 实体类会在一个persistence.xml的文件中被定义。但在Spring Boot框架中，并不需要使用到这个文件，被Entity Scanning实体扫描的方式所取代。

默认方式下，Spring Boot会查找在主配置类（使用@ EnableAutoConfiguration或@SpringBootApplication的主类）下所有的包，在这些包里面使用@Entity、@Embeddable或@MappedSuperclass的类将会被处理。

SPring Data JPA Repositories

Spring Data JPA repositories 通常是一组定义访问数据的接口。

根据你所定义的方法名称，JPA 查询会被自动的建立与执行。例如，CityRepository接口会声明一个findAllByState(String state)方法，根据state参数去查找所有的city。

对于更加复杂的查询，你可以在查询方法上使用@Query注解。

Spring Data repositories通常继承Repository或CrudRepository接口，如果你使用自动配置，Spring Boot 将会查找在主配置类（使用@ EnableAutoConfiguration或@SpringBootApplication的主类）下所有的repositories类。

创建与删除 JPA 数据库

默认情况下，只有在使用内嵌数据库（H2, HSQL或Derby）时，JPA数据库才会被自动创建。可以使用 spring.jpa.* 属性显式的设置JPA，比如，将以下配置添加到 application.properties 中可以创建和删除表：

spring.jpa.hibernate.ddl-auto=create-drop

运行结果

浏览器内输入localhost:8080/users/all链接后，因为此时数据库内无数据，此时返回[]

浏览器内输入localhost:8080/users/save?name=linj2n&age=18&gender=male后，返回success，save(Long id)方法测试通过。

浏览器内输入localhost:8080/users/find?id=1查找刚刚插入数据，返回结果正确，findOne()测试通过。

与第一步类似，插入一条User数据后，输入localhost:8080/users/all查询所有数据，显示正确，findAll()方法测试通过。

浏览器内输入localhost:8080/users/delete?id=1删除第一条数据，返回success字符串，删除成功。

再次输入localhost:8080/users/all查询所有结果。显示结果不包含删除的数据。测试通过。
