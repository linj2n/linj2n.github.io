Spring学习总结（3）Spring Boot + Spring MVC 入门实例

Posted on 2018/05/18

上一篇介绍一个使用 Spring 框架开发的登录模块，接下继续利用这个登录模块的业务逻辑，不同的是使用 Spring Boot 框架替代 Spring 框架，目的为了大致了解 Spring Boot 开发的一般步骤以及 Spring Boot 的优点。例子的完整代码已托管到 gayhub 上，地址https://github.com/linj2n/spring-demos

Spring boot好处

回顾前一篇使用 Spring 框架开发的登录模块的步骤，最多的一个步骤就是配置各种 xml 文件。Spring boot的出现目的是为了减少开发人员编写复杂的 XML 配置文件，仅通过几行代码就能实现一个可运行的 Web 应用，使得开发人员更多的关注业务逻辑开发。

创建 Maven 项目

配置pom.xml

  • 与使用 Spring 框架开发项目的需要在 pom.xml 文件中引入大量的 Spring 子模块依赖不同，这里的 pom.xml 只引入了一个 spring-boot-starter-web 启动器依赖，该依赖封装了 spring-web、spring-webmvc、jackson-databind 等模块依赖。在 Maven Projects 文件视图中 Dependencies 中可以查看相关依赖。

项目文件结构

spring boot 建议的目录文件如下。
根目录：com.example.myproject
项目目录:

|- com
    |- example
       |- myproject
            |- dao      // 存放数据访问类
            |
            |- domain   // 存放领域类
            |
            |- service  // 存放业务类
            |
            |- controller       // 存放页面访问控制控制类
            |
            |_ Application.java  // 主要用于做一些框架配置

持久层

Spring 框架提供了几种可选的操作数据库方式，有内置轻量级的 JdbcTemplate ，也有第三方持久框架 MyBait 或 Hibernate。这里我们使用上一篇持久层使用到的轻量级的 JdbcTemplate。

配置数据库连接信息

为了使 Spring Boot 能够自动装配数据源的连接，在资源根目录 resources 下创建一个 application.properties，在该文件加入数据库源连接信息。

配置数据库连接方式有二：

  1. 自定义连接方式
  2. JNDI方式

我们采用第一种方式，即使用自定义连接的方式

spring.datasource.name=helloworlddb
spring.datasource.url=jdbc:mysql://localhost:3306/helloworlddb
spring.datasource.username=root
spring.datasource.password=your-password
spring.datasource.driver-class-name=com.mysql.jdbc.Driver
