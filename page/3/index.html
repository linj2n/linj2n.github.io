为什么需要Session管理？

HTTP 其实是无状态的，即不同的请求没有上下文关系，但是很多时候页面上的操作是需要依赖上下文的，比方说登录注册购物车。这时候就是 Cookies 与 Sessions 登场的时候了。

HTTP协议是无状态，即不同的请求没有上下文关系，而在Web页面上操作一般是需要依赖上下文的，例如购物车记录商品，登录与注册。此时 Session 管理技术就出现了。

用于保持状态的四种方法有：

  • URL重写
  • 隐藏域（Hidden Fields）
  • cookie
  • HttpSession 对象

Servlet/JSP知识小结（2） JSP

Posted on 2018/01/05

JSP技术

JSP 全称JavaServer Page，是对Servlet技术的补充，可以将常规的、静态的HTML与动态生成的内容混合起来。

Servlet的缺陷与JSP的优点

Servlet 擅长于业务逻辑或数据处理，而在需要生成输出的时候，Servelt有几个缺陷无法克服：

  • Servlet 中的HTML的编写和维护十分困难
    在Servlet中编写HTML，必须使用大量PrintWriter.println()语句，对字符串进行拼接，当页面比较的复杂时，编写起来十分的困难。另外使用这种方式，所有文本和HTML标签必须硬编码，一旦对表示层做微小的该改动时，例如，修改背景颜色，也需要重新编译。
  • 非Java开发人员难以处理这些HTML
    将HTML嵌入到Servlet中，不了解Java语言的Web开发人员难以检查和更改这些HTML。

JSP的出现是为了拟补Servlet表示上的补足，使用JSP有以下几个优点：

  • JSP中的HTML的编写和维护更加的简单。
  • 可以使用标准的网站开发工具，例如HTML工具Dreamweaver。
  • 将静态HTML和动态内容进行了分离。

JSP生命周期

  • 转换阶段
    这个阶段只出现在页面创建或者页面被修改后，客户端的第一次请求。此时JSP容器将JSP页面转换成Servlet，这种转化只是简单地将所有模板文本改用println()语句，并且将所有的JSP元素转化成Java代码。此时的JSP文件为someJSP.java

  • 编译阶段
    这个阶段同样只出现在页面创建或者页面被修改后，客户端的第一次请求。JSP容器编译servlet源文件，生成servlet类。此时JSP文件为someJSP.class

  • 加载阶段
    客户端对JSP页面进行第一次请求时，JSP容器会将编译好的class加载服务器的内存中，并且常驻内存，直至JSP页面被修改。

  • 初始化阶段
    该阶段紧跟在加载阶段之后，同样也发生在客户端对JSP页面进行第一次请求的时候，JSP容器将调用jspInit()，进行初始化工作。

  • 执行阶段
    当每次客户端发出请求时，常驻服务器内存的servlet实例将调用_jspService()处理来自客户端的请求，并生成相应的响应格式，如HTML，返回给客户端。

  • 销毁阶段
    调用与JSP对应的servlet实例的销毁方法，然后销毁servlet实例；

什么是 Servlet？

Servlet是Server applet 的缩写，意思是运行在服务端的Java程序。

从作用上看，Servlet是一个中间层，负责接受来自Web浏览器或其他HTTP客户程序的请求，连接HTTP服务器上的数据库或应用程序，处理这些请求并返回响应。更具体一点：

  • 读取客户端发送的显式数据和隐式数据：显式数据有可能来自用户提交的HTML表单信息。隐式数据通常指的是HTTP信息，包括Cookie、浏览器所能识别的媒体类型等。
  • 生成结果：生成结果的过程可能需要访问服务端的数据库，也有可能需要调用Web服务。
  • 给客户端发送显式数据和隐式数据：这里的显式数据通常指文档，文档的格式是多样的，包括文本（HTML或XML）、二进制图（GIF图）等，最常用的格式是HTML
