【JavaScript语言精粹】读书笔记

Posted on 2018/06/20

1. Good Part

1.1 Why JavaScript

  • Web 已经变成了一个重要的应用开发平台，而 JavaScript 是 Web 浏览器的语言，是世界上最被轻视的语言之一。很大部分的原因在于浏览器的 API 和 DOM 内容的糟糕导致 JavaScript 语言被误解。
  • 拥有极强表达能力的语言

1.2 Analyzing JavaScript

JavaScript 好的一部分：

  • 函数、弱类型、动态对象与对象字面量表示法

JavaScript 差的一部分：

  • 全局变量的编程模型

函数

  • JavaScript 的函数是基于词法作用域的顶级对象。即函数是根据词法来划分作用域的，而不是动态地划分作用域。

  • JavaScript 是第一个称为主流的 lambda 语言。

弱类型

  • 大部分编程语言都流行要求强类型，原理在于强类型允许编译器在编译时检测错误，越早检测和修复错误，付出的代价越小，但事实上强类型并不会让你的测试工作轻松。

  • 弱类型比较自由，无序建立复杂的类层次。

对象字面量表示法

  • JavaScript 通过列出对象的组成部分，因此能够简单地创建一个对象。
  • 这种表示法促使了 JSON 格式的产生。

原形继承

  • 原型继承是 JavaScript 中的一个有争议的特性。即JavaScript 有一个无类别（Class-free）的对象系统。
  • 对于习惯使用类去创建对象的程序员来说，需要习惯原型继承的特性。

2.1 White Space

  • 语法图
  • JavaScript 提供两种注释，一种是/* */包围的块注释，另外一种是//行注释。
  • 建议：避免使用/* */注释，用//注释代替它。

2.3 Numbers

  • 语法图
  • 仅有一个单一的数字类型，内部被表示为 64 位的浮点数。
  • 与 Java 的 double 不一样，并没有分离出整数类型，因此 1 与 1.0 是相等的。
  • NaN是一个数值，表示一个不能产生正常结果的运行结果，NaN不等于任何职，包括它自己。
  • Infinity表示所有大于1.79769313486231570e+308的值

2.4 Strings

  • 语法图
  • JavaScript 没有字符类型，要表示一个字符，只需创建仅包含一个字符的字符串即可
  • 字符串是不可变的，一旦字符串被创建，就无法改变它，但能通过+运算符去连接其他的字符串。

2.5 Statements

  • 下面的列出的值被当做是false，其他的所有值都被当做true

    • flase
    • null
    • undefined
    • 空字符串
    • 数字 0
    • 数字 NaN

  • throw 语句跑出一个异常，若 throw 语句在一个 try 代码块中，name控制权会调到 catch 从句。若 throw 语句在函数中，则该函数调用会被放弃，且控制权会调到调用该函数的 try 语句的 catch 从句中

  • 不允许在 return 关键字和表达式之间换行

  • 在 web 浏览器中，每一个<script>标签都提供一个被编译且立即执行的便一单元，因为缺乏连接器，JavaScript 把它们丢进了一个公共的全局名字空间中。

Object

  • JavaScript 的对象包括简单类型与对象。简单类型有：数字、字符串、布尔值、null 与 underfined，其他所有的值都是对象。
  • 对象是属性的容器，属性拥有名字与值。属性的名字可以包括空字符串在内的任意字符串。属性值可以是出 underfined 值之外的任何值。
  • 对象是无类别（class-free），因此对新属性的名字和值没有约束。
