Spring学习总结（1） Spring 概述与入门配置

Posted on 2018/05/08

Spring概述

Spring是什么

在我们的传统的应用开发中，一个完整的Web程序是由一组相互协作的对象组成，共同完成所需的功能。除了开发业务逻辑，为了使得我们的程序变得低耦合、高内聚，我们要花大量的时间在创建以及管理对象之间的依赖关系上面。

我们是否能够找一种方法能够帮助我们管理这些对象之间的依赖关系，简化我们的开发呢？

这个时候，设计模式出现了，在一定程度上能简化我们的对对象及类的管理。例如“工厂模式”、“生成器模式”，可以帮助我们创建、管理对象之间的依赖关系。可问题又来了，使用设计模式虽然方便了管理，但引入了新的类——工厂类、生成器类。等于说除了一般的类，我们需要设计出其他额外的类来管理这些一般类，增加了我们的负担。

如果能有种通过配置方式来创建对象，管理对象之间依赖关系，我们不需要通过工厂和生成器来创建及管理对象之间的依赖关系，这样我们是不是减少了许多工作，加速了开发，能节省出很多时间来干其他事。Spring框架刚出来时主要就是来完成这个功能。

功能

Spring框架除了能帮我们管理对象及其依赖关系外，还能提供以下的功能：

  • 支持面向切面编程的功能
    • 日志记录
    • 权限控制
    • 性能统计
  • 管理数据库事务
  • 可以与第三方框架无缝集成

总的来说，Spring能够完成除了业务开发之外的所有的事情。

核心

IoC/DI 思想

我们上面提到Spring的一大好处是可以为我们管理对象及其依赖关系，Spring如何完成这样的功能的？

核心就是 IoC/DI思想，即Inversion of Control，缩写为IoC，控制反转还有一个名字叫做依赖注入（Dependency Injection），就是由容器控制程序之间的关系，而非传统实现中，由程序代码直接操控。

上面的解释比较晦涩，举个与传统开发对比的例子或许能更快的理解什么是控制反转。

传统的 Java 程序开发：当需要一个对象时，我们直接在程序内部通过 new 的方式创建对象，这里是程序主动创建依赖对象（正转），程序控制对象的创建。

引入IoC/DI容器后：当需要一个对象时，有一个专门的 IoC 容器来帮忙创建对象，程序被动接受依赖对象（反转），IoC/DI容器控制对象的创建。

Servlet/JSP的集成：MVC 架构

实现 MVC 模式的应用程序由三个模块组成

  • 模型 Model： 负责数据和业务逻辑。
  • 视图 View： 负责应用程序的显式
  • 控制器 Controller： 负责接收用户的输入，并控制 Model 和 View 做出相应的改变

控制器 Controller

在 MVC 框架中，控制器 Controller 是由一个 Servlet 或者过滤器实现的，例如常用的 Web 框架 Struct 1 与 Spring MVC 在它们的 MVC 架构中使用一个 Servlet Controller， 而另一个流行的框架 Structs 2，则是使用过滤器。

视图 View

视图 View 层一般由 JSP 技术实现，例如 Spring MVC 支持许多类型的视图用于不同的表示技术，其中最常用就是 JSP 技术。

模型 Model

模型 Model 可以由 JavaBean、 EJB、 POJO(Plain Old Java Object)实现，其中 POJO 与 JavaBean 比较流行。

在 MVC 模式中的应用程序，每一个 HTTP 请求都必须被定向到控制器中，请求的 URI 告诉控制器要调用那一个 Action，Action 指的是 Web 程序能够执行的一箱操作，与 Action 相关的 Java 对象被称为 Action 对象，一个 Action 类可以服务几个不同的 Action，例如 Structs 2 和 Spring MVC ，或单个 Action ，例如 Struct 1。

Servlet/JSP学习总结（6）Filter 过滤器

Posted on 2018/02/09

Filter 顾名思义为过滤器，无论客户端向服务端通过 URI 请求资源（JSP、Servlet、静态图片等），还是服务端响应客户端的请求，都必须经过实现了 Filter 接口的 filter 实例。

例如，在 filter 对象中通过HttpRequest.setCharacterEncoding()方法修改请求信息的编码方式。又如可以对请求的 URI 信息判别，使得请求转向其他的 JSP 页面，框架Struct 2 的控制器就是通过过滤器实现的。

总的来说，Filter 技术使得管理 Web 资源变得十分便利。不少 Java Web 高级技术都是建立在 Filter 之上，因此了解 Filter 这一种重要技术是十分必要的。

Servlet/JSP学习总结（4）EL表达式

Posted on 2018/02/07

EL 全名为Expression Language。EL主要作用：

  1. 获取数据
    EL表达式主要用于替换JSP页面中的脚本表达式，以从各种类型的web域中检索java对象、获取数据。(某个 web 域中的对象，访问 Javabean 的属性、访问 List 集合、访问 Map 集合、访问数组)
  2. 执行运算
    利用EL表达式可以在JSP页面中执行一些基本的关系运算、逻辑运算和算术运算，以在JSP页面中完成一些简单的逻辑运算。${user==null}
  3. 获取web开发常用对象
    EL 表达式定义了一些隐式对象，利用这些隐式对象，web开发人员可以很轻松获得对web常用对象的引用，从而获得这些对象中的数据。
  4. 调用Java方法
    EL表达式允许用户开发自定义EL函数，以在JSP页面中通过EL表达式调用Java类的方法。

概念

名称 解释 示例
标签 HTML是由一套标记标签 （markup tag）组成，通常就叫标签，标签由开始标签和结束标签组成
元素 元素指的是从开始标签到结束标签之间所有的代码，属性用来修饰标签的 <h1 align="center">居中标题</h1>
标题 标题会自动粗体，大写其中的内容，并且带有换行效果 <h1>标题1</h1>
段落 <p>标签即表示段落是paragraph的缩写，自带换行效果 <p>段落4 </p>
粗体 <b> <strong>b是bold的缩写，仅仅表示该文本是粗体的，并不暗示这段文字的重要性。strong虽然也是粗体，但是更多的是强调语义上的加重，提醒用户该文本的重要性。在SEO（搜索引擎优化）的时候，也更加容易帮助用户找到重点的内容 <b>，<strong>
斜体 <i>和<em>都可以表示斜体效果，i是italic的缩写，仅仅表示该文本是斜体的，并不暗示这段文字的重要性，em 是 Emphasized的缩写，虽然也是斜体，但是更多的是强调语义上的加重，提醒用户该文本的重要性。 常常用于引入新的术语的时候使用。
预格式 有时候，需要在网页上显示代码，比如java代码就需要用到<pre>
下划线 <ins>即下划线标签，但是通常来讲，不要给普通文本加下划线，因为用户会误以为是一个超链
删除线 <del>即删除标签，delete的缩写
