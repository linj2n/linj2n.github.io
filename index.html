#399.Nuts and bolts problem

###来源：lintcode 399. Nuts & Bolts Problem

##Description：
Given a set of n nuts of different sizes and n bolts of different sizes. There is a one-one mapping between nuts and bolts. Comparison of a nut to another nut or a bolt to another bolt is not allowed. It means nut can only be compared with bolt and bolt can only be compared with nut to see which one is bigger/smaller.

We will give you a compare function to compare nut with bolt.

##Example
Given nuts = ['ab','bc','dd','gg'], bolts = ['AB','GG', 'DD', 'BC'].

Your code should find the matching bolts and nuts.

one of the possible return:

nuts = ['ab','bc','dd','gg'], bolts = ['AB','BC','DD','GG'].

we will tell you the match compare function. If we give you another compare function.

the possible return is the following:

nuts = ['ab','bc','dd','gg'], bolts = ['BC','AA','DD','GG'].

So you must use the compare function that we give to do the sorting.

The order of the nuts or bolts does not matter. You just need to find the matching bolt for each nut.

##Input and output analysis: ###题目大意： 给定`String[] nuts`和`String[] bolts`分别模拟螺帽和螺丝集合，这些集合里面的元素有大小之分且没有重复的元素。`nuts[]`和`bolts[]`一一对应，即`nuts[]`中的任意元素一定能在`bolts[]`与其匹配的元素，反之亦可。函数需要对字符串数组进行处理，使得最后两个数组满足条件：

对题意的理解还需要注意一下几点：

  • 满足输出条件不一定要求两数组有序，只要同样下标的元素一一对应即可。

    如nuts[] = {N1,N2,N3,N4}; bolts[] = {B1,B2,B3,B4}; // 满足要求
    又如nuts[] = {N2,N1,N4,N3};bolts[] = {B2,B1,B4,B3}; // 满足要求

  • nuts和bolts中字符串元素是否匹配，与字符串的值无关、与ASCII码无关。也就是说对nuts或bolts按照字符串面值大小排序是无意义的。

  • nuts和bolts中的元素是有大小的之分的，但不可以通过比较两个螺丝得到这两个螺丝的大小，同样也不能通过比较两个螺帽得到这两个螺帽的大小，这也对应着上一条的限制。
  • Compare.cmp(String a, String b)函数的参数顺序不能颠倒，即a只能是nuts数组的元素，b只能是bolts数组的元素，参数顺序不一致会返回2。

##Approach#1 Brute force

###算法思路：
暴力破解法是我们很自然而然想到的方法。算法过程也很简单，利用两层循环，外层遍历nuts[]数组每一个元素，内层循环遍历bolts[]为每一个nuts[]找到与其匹配的元素，并把它放在相应的位置上，具体看代码。

###实现代码


public class Solution {
    public void sortNutsAndBolts(String[] nuts, String[] bolts, NBComparator compare) {
        // write your code here
        int n = nuts.length;
        for (int i = 0; i < n; i++){
            String currNut = nuts[i];
            for (int j = i; j < n; j++ ){
                if (compare.cmp(currNut,bolts[j]) == 0){
                    exch(bolts,i,j); break;
                }
            }
        }
    }
    public static void exch(String[] a, int i, int j){
        String temp = a[i]; a[i] = a[j]; a[j] = temp;
    }
};

##Approach#2 Customized Quicksort algorithm.

###算法思路：
通过题意我们可以知道，两数组的元素一一对应，并且有大小之分，当nuts和bolts按照升序或者降序排列时，同一个位置i，nuts[i]和bolts[i]是一一匹配的，即满足Compare.cmp(nuts[i],bolts[i]) == 0;。自然地，我们想到若能对nuts[]和bolts[]排序，问题便可以得到解决，并且使用一些良好的排序也可以在O(nlogn)内解决问题。

由于不能直接对数组nuts[]或bolts[]里面的元素相互比较排序，只能使用比较器拿出一个nuts[]元素与bolts[]元素比较（反之亦可）。所以归并排序、插入排序并不是很适用，剩下只能考虑快速排序。

这里复习一下快速排序，快速排序中的partition算法有一个很好的优点：

  • 每次通过选择一个pivot元素将数组划分两部分：
    a[0..pivot] <= a[pivot] <= a[pivot+1..n]
    一次partition结束后，a[pivot]被分配到了正确的位置，即a[pivot]的元素在之后的partition中不再移动。

给我们的启示是，使用partition算法，在bolts[]中取出一个元素Bi与nuts[]元素一一进行匹配、比较大小和划分。划分完成后，用nuts[]中找到元素Ni反过来再对bolts[]进行匹配划分。例如：

对于两个数组进行一次partition:
    nuts[]: N5,N8,N7,N4,N3
    bolts[]:B4,B7,B3,B5,B8
1.以B5作为要匹配的pivot元素，对nuts[]进行划分，一次划分之后：
    nuts[]: N4,N3,N5,N7,N8
    bolts[]:B5,B7,B3,B4,B8    // 比N5小的元素排在了N5的前半部分，N5的后半部分均比N5大，完成对nuts的一次划分
2.反过来取出N5，对bolts[]进行划分
    nuts[]: N4,N3,N5,N7,N8
    bolts[]:B3,B4,B5,B8,B7    // 完成对bolts的一次划分

从例子可以看出，当进行完两次的partition后，一个pivot元素就匹配完成，接下来再对左右两边的子数组递归处理完成两个数组的排序工作，排序完成后，两个数组的所有元素便一一对应了。

###实现过程注意的问题：

  • 使用compare.cmp(String a,Stringb)进行元素匹配时注意：从nuts[]取出一个元素划分bolts[]和从bolts[]取出一个元素划分nuts[]两种情况下需要调整cmp(..)参数顺序，并且要注意compare.cmp(..)的返回值对划分数组的影响。

    • partition(nuts,bolts[lo],lo,hi,compare);

    • partition(bolts,nuts[pivot],lo,hi,compare);

###实现代码


public class Solution {
    public void sortNutsAndBolts(String[] nuts, String[] bolts, NBComparator compare) {
        Collections.shuffle(Arrays.asList(nuts));
        Collections.shuffle(Arrays.asList(bolts));
        sortNutsAndBolts(nuts,bolts,0,bolts.length - 1, compare);
        // write your code here
    }

// pre：nuts[lo..hi]，bolts[lo..hi]内的元素未匹配。
// post：nuts[lo..hi]，bolts[lo..hi]内的元素已匹配。
public void sortNutsAndBolts(String[] nuts, String[] bolts, int lo, int hi, NBComparator compare)
{
    if (lo >= hi) return;
    int pivot = partition(nuts,bolts[lo],lo,hi,compare);
    partition(bolts,nuts[pivot],lo,hi,compare);    // 交替划分量数组，返回pivot下标位置
    sortNutsAndBolts(nuts,bolts,lo,pivot-1,compare);
    sortNutsAndBolts(nuts,bolts,pivot+1,hi,compare);
}

// pre: NA
// post: a[]中下标i、j两元素交换位置。
public void exch(String[] a, int i, int j)
{
    String temp = a[i]; a[i] = a[j]; a[j] = temp;
}

// pre:    未划分的字符串数组
// post: 对于返回值j，a中的元素满足：a[lo..j-1] < a[j] < a[j+1..hi]
// 这个partition算法实现是基于hoare版本的简要改写，虽不够简洁，但是易于理解和纸笔模拟，也可以使用其他的版本实现。
public int partition(String[] a, String pValue, int lo, int hi, NBComparator compare)
{
    // 找出与pValue匹配的元素，调整到lo位置上
    for (int i = lo; i <= hi; i++)
    {
        if (compare.cmp(a[i],pValue) == 0 || compare.cmp(pValue,a[i]) == 0)
        {
            exch(a,lo,i); break;
        }
    }
    // 只含有两个元素时，单独处理
    if (hi - lo == 1)
    {
        if (compare.cmp(a[hi],pValue) == -1 || compare.cmp(pValue,a[hi]) == 1)
        {
            exch(a,lo,hi); return hi;
        }else
            return lo;
    }

    // 以a[lo] 作为pivot元，划分数组
    int i = lo + 1, j = hi;
    while (true)
    {    
        // pValue与a[i]匹配，移动i下标，a[i] > a[lo]时停止。
        while (true)
        {
            if (i >= hi || (compare.cmp(a[i],pValue) == 1 || compare.cmp(pValue,a[i]) == -1))
                break;
            else
                ++i;
        }

        // pValue与a[j]匹配，移动i下标，a[j] < a[lo]时停止。
        while (true)
        {
            if (j <= lo || (compare.cmp(a[j],pValue) == -1 || compare.cmp(pValue,a[j]) == 1))
                break;
            else
                --j;
        }
        // i与j不交叉或相遇时，交换a[i]与a[j]
        if (i >= j)
            break;
        else
            exch(a,i,j);
    }
    exch(a,lo,j);   // 将a[lo]移动到中轴pivot位置，划分完成
    return j;
}

};
